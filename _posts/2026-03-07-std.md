## NAT 에 대하여

NAT 는 Network Address Translation 의 약자로, IP 주소를 다른 IP 주소로 변환하는 기술이다.
작은 의미로는 내부의 IP 주소를 묶어 외부용 IP로 변환하는 기술이다.
인터넷상의 IP 주소 볼륨을 줄이기 위한 방법이다. 
개인용 내부 IP를 다 인정해서 교환하려면 IP주소 크기가 한참 모자라기 때문.
변환 전-후 주소쌍을 NAT 테이블에 관리한다. 

- Static NAT : 내부 IP - 외부 IP 간 1대1 변환한다. 

- NAPT : P는 Port를 의미. 여러 내부IP를 하나의 외부IP로 교환하는 1:n 교환 방식이다.
         하나의 외부 IP에 여러 내부 IP가 묶여 있다면 각각을 어떻게 구분할까?
         바로 Port 번호를 다르게 부여하는 것.

- CGNAT : Carrier Grade NAT. ISP 에서 사용할 수 있도록 확장한 NAPT. 


## CGNAT 에 대하여

### 포트 할당
CGNAT는 대규모의 사용자로부터 IP를 받아 IP주소를 축약하기 때문에, 그 양이 어마어마 하다.
64512개의 포트번호만으로 바인딩하기에는 어려움이 있다.
그 해결책으로, 여러 외부 주소를 IP 주소 테이블로 분할하고, 각 주소마다 포트번호를 부과해 처리량을 분산한다.

포트를 할당하는 방법은 크게 세 가지다.

- 정적 할당 : 가입자에게 할당할 포트 번호 미리 정해줌 -> 로그 출력량 낮음, 집약효율 낮음
- PBA : Port Block Allocation -> 포트를 일정 구간 단위 블록으로 지정해 동적으로 할당 -> 로그 중간, 집약효율 중간
- 동적 할당 : 가입자에게 모든포트번호를 동적할당 -> 집약효율 최상, 로그 매우 많음(사용자의 정보 예측이 안돼서 다 기록해야 함)

### 커넥션 리미트

한 가입 단말이 너무 많은 포트를 사용하지 못하도록 수를 제한하는 기술.

### NAT Traversal

NAT 를 사용하면 마주하는 문제점이 있다.
NAT 테이블에 내부 IP - 외부 IP 쌍이 기록되려면, 내부 단말에서 송신지 IP를 가지고 외부로 패킷을 전송하는 과정이 있어야 한다.
즉, 내부 단말에서 송신을 하지 않았을 때 외부에서 패킷을 보내면, 그 패킷은 일치하는 IP주소를 NAT 테이블에서 찾지 못해 전부 버려진다.

이 문제점을 해결하기 위한 기술들이 있다.

1. 포트 포워딩 : 라우터에서 특정 IP주소+포트번호에 통신이 오면 지정된 내부 단말로 전송하는 기능.
2. UPnP (Universal Plug and Play) : 단말이 자동으로 포트포워딩을 요청하는 기능. 동적 포트 포워딩이 진행.
3. STUN (Session Traversal Utilities for NATs) : UDP 홀 펀칭. 단말이 외부로 UDP 패킷을 송신하면 그 세션으로 외부->내부 통신이 일정시간 허가됨. 이를 기반으로 양측 단말이 서로 STUN 서버랑 UDP 통신, 글로벌 IP와 포트번호를 교환해 직접통신을 시작하는 방법이다. 
4. TURN (Traversal Using Relay around NAT) : UDP 를 통한 직접 통신은 아니고, 그냥 TURN 서버를 중간에 두고 데이터를 교환하는 방식.

### 포트 매핑/필터링에 따른 NAT 분류

NAT의 동작은 크게 두가지 정책으로 구성된다.

1. Mapping : 내부 주소가 NAT에 매핑되었을 때, 외부 목적지가 달라도 같은 매핑을 유지할 것인가?
2. Filtering : NAT 매핑이 존재할 때, 어떤 외부 호스트라도 접근할 수 있는가?

이 두가지에 따라 Full, Restricted, Port-Restricted, Symmetric NAT가 구분된다.

매핑의 경우, EIM (Endpoint Independent Mapping), Address-Dependent Mapping 으로 나뉜다.

EIM 은 내부 주소를 NAT에 매핑하면, 패킷의 DEST 가 달라도 같은 매핑을 유지한다.
ADM 은 각 목적지마다 다른 매핑을 생성해야 한다.


필터링의 경우, EIF (Endpoint Independent Filtering), Address dependent, Address+port dependent 구분으로 나뉜다.

EIF 는 어떤 외부 IP라도 NAT의 IP교환을 통해 내부로 송신 가능하다.
Address dependent 방식은, **처음 통신할 때 목적지로 선택한 IP 에서 보내는 패킷** 만 필터링한다. 1.1.1.1 Dest 로 보냈다면 1.1.1.1 src 인 패킷만 받는 것.
Address+Port dependent 는, 목적지로 선택한 IP+Port 까지 구분한다. 매우 까다롭다.



이 두 정책을 조합해서 NAT를 4개로 분류한다.
| NAT 타입        | Mapping           | Filtering    |
| --------------- | ----------------- | ------------ |
| Full cone       | EIM               | EIF          |
| Restricted cone | EIM               | Address      |
| Port-restricted | EIM               | Address+Port |
| Symmetric       | Address dependent | Address+Port |


Symmetric NAT 의 경우 TURN 서버가 필요하다.
