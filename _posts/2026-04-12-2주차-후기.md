### 강의 후기

 1주차의 환경 세팅은 학습을 시작하기 위한 밑작업이었다.
 코드를 짜겠다면서 스스로 코드를 짤 환경을 구성하지 못하는 것은 우스운 일이다.
 라는 강사쌤의 강한 주장에 동의하는 바다.
 1주차에는 리눅스 환경을 설치하고 다 미는 걸 매번 반복했다. 
 모든 연결이 다 되었을 때 과감하게 OS까지 밀어버리고 다시 설치했다.
 수업과는 별개로 익숙해지고 싶어서 한 행동이었는데, 큰 도움이 되었다고 생각한다.

 그 과정에서 이전에 사용했던 가상 OS의 이름을 그대로 재사용한 적이 있는데,
 SSH 의 known_host 에 이전 키가 등록되어 있어서 새 OS의 키를 사용하지 못하고 충돌이 일어났었다.
 .ssh 에 접근해서 키를 밀어버리는 법, 
 ssh 명령어를 사용해서 기존 키를 known_host.old 로 보내는 법 등 여러 처리법을 찾아볼 수 있었다.
 
 또, 경험이 많아지다 보니 주변 팀원의 PC에 문제가 발생했을 때 
 적극적으로 나서서 해결을 도와줄 지식을 쌓을 수 있었다.
 옆자리 팀원의 컴퓨터에 에러가 많이 발생해서 오히려 공부하는 데 도움이 되었다.
 OS 의 쉘에서 자체 ssh root@ip 로그인은 되는데 쉘 외부에서의 접속이 되지 않을 경우에는
 보통 외부로부터의 SSH 접속을 차단하는 설정이 켜져 있거나,
 root 계정으로의 접속을 차단하는 설정이 되어 있음을 알고 설정을 변경해 고칠 수 있게 되었다.
 
 -

 2주차에는 주로 파이썬과 코딩의 기초에 대한 수업이 진행됐다.
 대학에서 C, C++, Java 는 배웠었고, 그 후에 Rust도 어느정도는 다룰 수 있었다.
 그래서 파이썬은 수업 전에 몇 시간 정도 사전학습을 하니 대부분 아는 내용들이었다.
 
 하지만,
 수업에서 정말 중요했던 핵심은 "코드를 짜는 방법" 이었다.

 강사쌤은 근본주의자라고 해야 할까? 
 가장 기초적인 원리로부터 응용을 배워 나가는 것을 선호하신다.
 이는 평소 내 성향과도 딱 들어맞는 부분이라 재미있게 수업을 들을 수 있었다.
 그렇지만, elif 문을 쓰지 말고 if 문으로만 코드를 짜라는 말은 받아들이기 힘들었다.
 내가 if, else, elif 를 모르는 것도 아닌데 굳이 그렇게 해야 할까?
 심지어 문제도 너무 쉬워서 if문을 많이 쓰지 않아도 풀 수 있었다.
 그래서 if 문 수업 처음엔 내가 생각하는 가장 편한 방식으로 코드를 짰었다.

 쌤은 if문만 가지고 코드를 짜면 보이는 것이 있다고 계속 주장하셨고,
 나도 결국 한 번 속는 셈 치고 if만 10개 가량 써서 코드를 작성했다.
 그런데 해 보니까 무슨 말씀인지 이해가 됐다.

 if 문만 써서 코드를 짠다는 건 코드를 '가장 작은 단위' 까지 분리한다는 뜻이다.
 서로 상관관계가 없는 하나 하나의 조건들로 분리된 코드.
 손만 좀 아플 뿐 생각을 구현하기에는 매우 편한 방식이었다.
 그리고, 각 조건문들이 독립적이니까 같은 일을 하는 조건문끼리 모으기도 편했다.
 그 말인 즉, 코드를 기능 단위로 분리하기 매우 용이했다는 뜻이다.

 그때 내 말을 들은 쌤이 "그래, 기능 단위. 기능이 영어로 뭔데." 
 라고 하셨는데, 기능 = function = 함수임을 떠올리자 팔에 소름이 돋았다.
 별 것 아닌 내용 같은데..
 if문만 쓰라고 하는 이상한 요구 기저에 있는 철학을 엿본 것 같았다.
 
 그 전까지는 함수가 뭐냐고 물어본다면 
    "지정된 입력에 대해 원하는 출력을 제공하는 코드 뭉치"
 정도라고 생각했다.
 그런데 이 과정을 거치고 나니, 함수는 
    "하나의 기능을 제공하기 위한 코드들의 모음"
 이라는 생각이 들었다.

 여태껏 내가 작성한 코드들이 복잡했던 이유는 기능을 '명확하게' 구분하지 않아서였다.
 개념을 체득하고 나서 코드를 짜 보니 코드가 깔끔하고 읽기 쉬워졌다.
 심지어 난, 코딩을 하면서 발상의 전환만 생각해 왔었기 때문에,
 이 '최소 기능 단위로 코드 분리하기' 라는 능력치가 생각보다 부족했다!

 그래서 수업이 끝나면 집에 가서 GPT한테 문제좀 내 달라고 해서, 
 깔끔한 코드 구조를 만들어 보려고 노력했다.

 최소 단위 코드 분리 -> 패턴 찾기 -> 기능 단위로 모으기 -> 단순화하기.
 + 연속적 값을 단편화하기.
 + 주석을 달아 코드를 명료하게 하는 습관 들이기.
 + 선언부와 각 기능별로 구분하기.

 다른 사람들이 문법을 배울 동안 나는 최대한 좋은 코드를 짜는 습관을 익히기 위해 노력했다.
 그 결과 2주차까지의 수업은 매우 만족스러웠다.
 내일이 매우 기대되는 바다.
