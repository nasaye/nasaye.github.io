암호 : 
평문을 일정한 규칙으로 변환해 약속된 대상만 내용을 알수 있도록 보호하는 기술, 정보.

암호화 : 
encrypt, 인가된 대상 외에는 읽지 못하도록 평문을 변환.

평문 :
plaintext, P. 암호화 전 메세지

암호문 :
ciphertext, C. 암호화 후 메세지

D = 암호알고리즘, K = 키.

C = E(K,P)
P = D(K,C)

---

암호화 보안 상식

1. security by obscurity :
암호 알고리즘을 비밀로 해서 보안 유지하려는 행위,
숨김에 의한 보안.
절대 하지 말 것. 폭로되면 끝이다.

2. 약한 암호가 암호화 안한 것보다 위험

3. 어떤 암호라도 언젠가는 해독된다.
(해독까지의 비용이 평문의 가치에 비해 무거운 밸런스 맞추기)

4. 암호는 보안의 일부분에 불과하다.
암호 알고리즘이 아무리 탄탄해도 시스템의 다른 링크(사회공학)
이 약하면 쉽게 뚫리기 마련이다.

치환 Substitution :
값을 다른 값으로 대체한다.

전치 Transposition :
값의 위치를 섞어 원래 의미를 감춘다.
자리를 바꾸는 규칙.

키 공간:
사용할 수 있는 모든 키의 집합(경우의 수)

키 비트수:
키공간 표현을 위한 비트 수

8비트를 전치할 때
8개의 값을 순열하는 경우의 수 = 8!
8비트 전치 암호화의 키공간은 40320. 
-> 표현하려면 16비트 필요.

8비트를 치환하는 경우의 수
8비트 경우의 수 256가지를 각기 다른 경우의 수(256)로 매칭.
즉 256*255*254...1 = 256!.
8비트 치환 암호화의 키 공간은 256!. 
표현하려면 log 2(2^8 !)
-> 1684비트 정도 필요. 너무 큼.

-> 보통 키를 쓰지 않고,
256!개중 1개 치환표로 골라 씀. 
256 * 8bit = 256Byte.

치환표 : 
8 bit 입력 -> 8 bit 출력.

S-Box :
내부 비선형적 치환이 되는 치환표.
혼돈. confusion . 
값을 비선형적으로 바꿔 평문,키->암호문 선형관계 깨트림.

확산과 혼돈

혼돈: 
평문-키-암호문 사이의 관계를 복잡하고 예측 불가능하게 만듬
= 평문과 키가 암호문에 미치는 관계의 선형성 파괴.
SB0X (Substitution)

확산:
평문의 한 비트 변화가 암호문 전체에 퍼지게 하는 것.
평문 패턴 / 통계적 공격 / 비트 영향 추적 막기.
Permutation, shift, rotate, MixColumns 등등


---

선형성:
입력들의 선형 결합이 출력에서도 그대로 유지되는 성질
선형 결합 = 보통 xor.

선형 공격 = 
많은 P-C 쌍 수집
->특정 선형식이 50%보다 조금 더 맞으면 편향추정.
->핵심은 틀린 키는 50%에 가까운 결과가 나옴.
->가장 벗어난 키 후보가 진짜 키일 확률 높음.

링크암호화, 종단간 암호화

링크 암호화 :
헤더까지 암호화 -> 패킷 라우터나 중간장비가 복호화.

종단간 암호화 :
헤더,트레일러 두고 페이로드만 암호화
앱 계층 암호화.
사용자가 메세지 암호화를 결정.
트래픽 분석에 취약.

대칭키 암호와 비대칭키 암호

대칭키 암호:
암호화 K  = 복호화 K.
즉 키의 양 사용자간 공유 필요
공개키보다 빠르고, 길이도 작다.

비대칭키 암호:
암호화 K != 복호화 K.
느리기 때문에 대칭키 암호화의 공유에 쓰임.

일방향 해시함수:
임의 길이 값에 대한 고정길이 출력
결정적 : 동입력 => 동출력
역상 저항성 : 역추적 X
충돌 내성 : 다른 값에는 거의 다른 결과.
--> 입력값의 무결성 검증에 사용. 
(야, 데이터 보내고 해시한거 보내봐. 나도 해시해보게. )

메세지 인증 코드 (MAC, Message Auth Code)
1. 메세지 M 을 보내고 싶음. 암호화는 딱히 필요 없음. 
2. 둘이 서로 비밀키 K 공유
3. f(K,M) => MAC (이상한 값.)
4. M에다가 MAC 딸려 보냄
5. M받은 놈이 f(K, 받은M) 해서 MAC 랑 똑같이 나오나 확인.

HMAC (Hash MAC)
길이확장공격 방지:
해시값만 알아도 원본데이터 뒤에 데이터 붙인 해시를 계산할 수 있는 공격.
SHA-256에 사용가능. 내부 상태를 이어서 계산하기 때문.
 H((K ⊕ opad) || H((K ⊕ ipad) || M))
그래서 opad, ipad 그냥 패딩 넣고 XOR, OR 등등 해시2번 등 해서 막는거.

디지털 서명
메시지 무결성 + 송신자 인증 + 부인 방지
 S = H(M)^d mod n
d = 개인키
 M, S 전송
 H(M) =S^e mod n
