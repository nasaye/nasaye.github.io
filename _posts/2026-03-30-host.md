### 자주 보이는 용어 호스트, 어떤 느낌일까?

Host, 네트워크에서도 서버에서도 자주 쓰이는데 이번엔 SSL에서까지 쓰이더라.
그런데 각 용처마다 미묘하게 뉘앙스가 달라서 정리해본다.

- 네트워크의 HOST : IP주소를 가지고, 네트워크에 참여하여 데이터를 주고받을수 있는 독립 기기

- 서버 HOST : 데이터를 요청하는 클라이언트에게 서비스를 제공하는 주체

- SSH Config의 HOST : 특정 규칙들을 적용할 대상의 이름



서버의 호스트는 집주인, 네트워크의 호스트는 손님, SSH의 호스트는 주소이름 같은데,
일관성 없어 보이는 항목들을 HOST는 어떻게 묶을까?

> 누가 누구를 품고 있는가, 누가 주체인가

네트워크 host 는 기기 자신이 주체임을 뜻하는 말이고,
서버의 host 는 서버가 주체가 되어 손님들의 요청을 받아준다는 뜻.
SSH의 host 는 실제 주소와 관련설정들을 관리하는 주체라는 느낌으로 사용한다.
가상화 에서는 내 윈도우가 VM을 관리하는 주체이니 윈도우가 호스트가 되겠다.
