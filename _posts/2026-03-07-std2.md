## 개요

IPv4 / IPv6 병용을 위한 기술들
ICMP (Internet Control Message Protocol)

### Dual Stack

하나의 단말에 IPv4, IPv6 둘 다 할당하는 기술.

양 쪽 프로토콜 단말에 모두 대응 가능하다는 장점, 운용 부하가 많이 걸린다는 단점.

### DNS64/NAT64

DNS 서비스를 이용해 IPv6 단말이 IPv4 단말과 통신하도록 하는 기술.

DNS64 
1. IPv6 단말이 DNS서버에 도메인주소의 IPv6 주소를 요청한다.
2. 해당 도메인주소가 IPv4만 지원한다면, 64:ff96::/96 프리픽스에 해당 IPv4주소 32비트를 뒤에 붙여서 반환한다.

NAT64
1. IPv6 단말은 전달받은 64:ff96::????/96 주소로 패킷을 보낸다.
2. NAT 장치가 수신지가 64::ff96::/96 인 패킷을 받으면 DNS64/NAT64 기술임을 파악하고 뒤의 32비트를 수신지 주소로 설정한다.
3. 그리고 송신지 IPv4  주소를 임의로 생성변환한다.


### 터널링 

IPv4 over IPv6
IPv6 over IPv4

두 단말간에 같은 IP프로토콜로 통신하는데 중간에 다른 프로토콜 라우터가 껴 있다면?
IPv4를 IPv6으로 캡슐화 해서 지나가는 (헤더를 붙여서) 방식으로 해결한다. 
이게 터널링.  A over B = A를 B로 캡슐화.

### ICMPv4
인터넷 제어 메세지 프로토콜로, IPv4와 반드시 구현되어야 함.
ICMP4 는 IPv4 의 헤더에 프로토콜 번호를 1로 기입하고, 4바이트의 필드를 (타입8비트, 코드8비트, 체크섬 16비트) 가진다.
페이로드에는 오류내용 등을 기록.

#### ICMPv4 동작

- 에코 요청 - 응답 (8 0,0 0)
  : 트러블 슈팅의 시작. 네트워크 계층 소통레벨을 ICMPv4에코로 시작한 후, reply 가 돌아오면 L4, 그다음 앱 계층으로 차근차근 확인.
  : 에코 릴레이가 돌아오지 않으면 L3, L2, L1 순으로 내려감.

- dest unreachable (3 n)
  : IP패킷을 수신지 단말까지 라우팅하지 못했을 때 발송하는 핑.

- TTL Exceeded(11 0)
  : 패킷 유효기간 경과 파기. 무한루프 방지, 통신경로 확인. (ex: counttoinfinite - RIP hop max limit)
  * 라우터 루프 예시 : pc-r1-r2-인터넷 으로 연결된 라우터 네트워크에서 r2의 디폴트게이트웨이가 r1으로 잘못 설정
                     : 결국 pc-r1-r2-r1-r2-r1-r2.. 루프에 빠짐. TTL 필요.

**traceroute / tracert : TTL Exceeded 응용, TTL을 1부터 하나씩 증가시키면서 반환듸는 ICMPv4를 보고 경로 파악.**

### ICMPv6 

IPv6 헤더에 타입,코드,체크섬. 
타입과 코드가 중요.
IPv4 에서의 ARP, IGMP 까지 전부 ICMPv6가 담당하기 때문에 중요.

#### ICMPv6 동작
기본적으로 v4랑 비슷. 에코 리퀘스트-릴레이, dest unreachable.
ARP를 대신하는 NDP (Neighbor Discovery Protocol) 가 중요.

- 주소 중복 감지 : IPV4의 GARP 가 담당하던 것.IP 주소 설정시 DAD(Duplicated Address Detection) 처리.
DAD : 단말이 네트워크 접속시 자동으로 더미링크주소 생성 
NS(neighbor solicitation) 135 로, ff02::1:ffxx:xxxx dest 로 멀티캐스트. 사용 가능한지 확인. 송신지는 ::/128. 더미주소 사용하려는 건 페이로드 부분의 목표 주소 필드에 삽입.
NA(neighbor Advertisement) 136 돌아오면 중복.

- dest IPv6 주소에서 dest MAC 주소 구하기 (ARP 기능)
NS 135 로 멀티캐스트 (목표 주소) 
NA 136 목표주소 일치하면 회신 유니캐스트.

- 네트워크 정보 제공
SLAAC나 DHCPv6 등 주소생성시에 133 RS 패킷으로 네트워크 정보 문의. 
라우터는 NAC, MTU, 프리픽스 등 134 RA 패킷에 넣어 반환.
