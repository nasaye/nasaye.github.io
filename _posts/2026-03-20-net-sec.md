## 필수 개념 정리

### 컴퓨터 보안
 
- 컴퓨터 보안 : 정보 시스템 자원의 기밀성, 무결성, 가용성을 보전하는 자동화된 보호 체계. 
- 네트워크 보안 : (인터넷 보안) 각 단말간의 네트워크에서 전송되는 데이터들을 보호하는 보호 체계.

### CIA Trid 
컴퓨터 보안의 핵심 목표.

CIA Trid + 2 : 기밀성, 무결성, 가용성, 인증성, 책임성

- 기밀성 : 인가된 규정, 절차, 사용자만 데이터에 접근할 수 있도록 보장.
    - 데이터 기밀성 : 정보가 부정한 사용자에게 노출되지 않도록.
    - 프라이버시 : 개인의 활동에 대한 메타데이터가 보호되도록.
- 무결성 : 적법하지 않은 사용자, 또는 시스템의 문제로 인해 데이터가 위.변조되지 못하도록 보장.
    - 데이터 무결성 : 규정, 허가된 상태에서만 정보, 프로그램 변경
    - 시스템 무결성 : 시스템의 의도된 기능을 손상, 조작 없이 온전하게 수행하는 것
- 가용성 : 시스템이 허가된 사용자에게 조작되지 않고 의도대로 잘 수행되도록 보장.
- 인증성 : 사용자, 전송과정, 메세지, 출처의 유효성을 보장.
- 책임성 : 시스템의 보안 책임이 있는 곳까지 로그를 기록하고 추적할 수 있도록 보장.


### OSI 보안구조
ITU-T X.800 에서 정의하는 보안 체계 접근법.

#### 보안 공격
보호받아야 할 데이터의 안전성을 침해하는 모든 행위.
- 소극적 공격 : 데이터에 대한 도청, 감시. 데이터를 조작하지는 않음.
    - 메세지 내용 갈취 : release of message, 전달되는 메세지의 내용을 보거나 취득 (기밀성)
    - 트래피 분석 : traffic analysis, 송수신 IP, 통신방법, 횟수 등 메세지의 유형, 빈도, 길이 등을 분석. (기밀성) 
- 적극적 공격 : 데이터를 수정,위조,변조하는 등의 직접적인 공격 행위.
    - 신분위장 : masquerade, 다른 신분인 척 위장하는 공격. 허가되지 않은 권한을 이용할 수 있음. (인증성, 추가로 기밀성 무결성 공격 가능해짐)
    - 서비스 거부 : Denial Of Service, 서비스가 제대로 운용되지 못하도록 공격. (가용성)
    - 재전송 : replay, 소극적 공격 등을 통해 얻은 데이터 단위를 재전송하여 인가되지 않은 효과를 노림. (인증성)
    - 메세지 수정 : modification of message, 메세지를 수정하여 인가되지 않은 효과를 노림. (무결성)

#### 보안 서비스
시스템, 데이터 전송의 보안을 제공하는 서비스로, OSI의 프로토콜 계층에 의해 제공.
보안 정책을 성립하고, 보안 메커니즘으로 인해 구현된다.

- 인증(Authentication) : 통신 주체가 적법함과, 통신과정에서 무결성이 지켜졌다는 것을 확인시켜 주는 것.
    - 대등개체(peer) : 상대방 신원 인증
    - 데이터 출처(data origin) : 데이터의 출처. 
- 접근 통제(Access control) : 누가, 어떤 조건하에, 어떤 자원을 사용할지 제한하는 것.
- 데이터 기밀성(Confidentiality) : 데이터의 불법적 노출 막음.
    - 연결형 : TCP~ 연결시 모든 사용자 데이터 보호 
    - 비연결형 : 단일 데이터 블록 단위 보호
    - 선택-필드 : 블록의 일부 필드 보호
    - 트래픽-흐름 : 트래픽 분석으로부터 보호 (트래픽 패딩 등)
- 데이터 무결성(Integrity) : 수신 데이터가 공격받지 않았음에 대한 확신.
    - 복구 가능 연결, 복구 없는 연결, 
    - 선택 필드 연결/비연결 : 선택된 필드에 대한 무결성 제공
    - 비연결 : 단일 비연결 블록 무결성 제공.
- 부인봉쇄(Nonrepudation) : 통신에 참여했던 사실을 부인하는 것을 방지.
    - 출처(Origin) : 메세지가 출처로부터 송신되었음을 증명
    - 목적지(Destination) : 해당 개체가 메세지를 수신했음을 증명

### 기본 보안 설계원칙 NCAE13

- 경제적 매커니즘(Economy of mechanism) : 보안수단의 규모 최소화 -> 취약점 면적줄이기, 관리 수월. 안 복잡.
- 기본 안전장치(Fail-safe defaults) : 접근불허를 기반으로 접근 허가 상황을 설정하라.
- 완전 중재(Complete meditaion) : 모든 접근은 접근 통제 매커니즘을 거쳐라.
- 개방 설계(Open Design) : 보안 메커니즘은 공개 설계하라. (다수의 피드백->신뢰)
- 권한 분리(Separation privilege) : 권한을 하나에 집중하지 말라. (하나 뚫리면 다털림) 
- 최소 권한(Least privilege) : 각각의 작업을 수행할 때 최소한의 권한만 허가하라. 
- 최소 공통 메커니즘(Least common mechanism) : 여러 사용자가 "구조,경로,로직,상태" 를 공유하는 매커니즘을 최소화.  -> 정보유출, 권한상승, 사이드채널 방지
- 심리적 수용성(Psychological acceptability) : 사용자를 너무 방해해 반발감이 들게 할 경우 실수가 발생.
- 분리 (Isolation) : 물리적으로는 중요정보를 공개정보와 분리, 논리적으로는 개별 사용자 프로세스,파일을 서로 분리, 보안 매커니즘 자체를 호스트 시스템으로부터 분리.
- 캡슐화(Encapsulation) : 데이터 객체 집합을 캡슐화하여 보호.
- 모듈식(Modularity) : 각각의 보안 기능을 보편적 모듈로 제공, 또는 새 보안 기술을 전체 시스템 변경 없이 쉽게 업데이트 하도록.
- 계층화(Layering) : 심층 방어. 깊이에 따른 다수의 보안 조치.
- 파격의 최소화(Least astonishment) : 사용자를 놀라지 않게 하라. 직관적이고 이해하기 쉽게.

### 공격 트리

 보안 사고를 루트 노드로 삼고, 그 사고를 달성할 수 있는 방법들을 서브 노드로 연결한다. 리프 노드는 공격을 시작할 수 있는 다양한 방법들을 표시.
보안 취약점 악용 기법들을 계층적 트리 구조로 나타냄으로써 공격 전략과 패턴에 대한 통찰력을 제공.

### 네트워크 보안 모델의 4가지 기초 임무

1. 공격자가 변환을 깰 수 없는 암호화 알고리즘 설계.
2. 알고리즘에 사용될 비밀 정보 생성.
3. 비밀 정보를 공유, 배분할 수 있는 방법 개발.
4. 보안 서비스를 위한 양 통신 주체간 프로토콜 구체화.
