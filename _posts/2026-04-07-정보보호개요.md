정리법 : 키워드, 1줄 설명, 시험포인



---

정보보호 목표 : 기밀성, 무결성, 가용성, 책임추적성, 인증성 (기무가인책)
기무가 = CIA TRID 

정보보호 관리대책 : 
기술적(암호,백업, 보안), 
물리적(화재예방, 출입통제 등), 
관리적(조직 제도,교육, 인적관리 등등).

---

OSI 보안구조
보안공격

기밀성 위협: 
스누핑 (도청)
트래픽분석(메타데이터,트래픽 정보) 
-> 정보 읽지만 변경은 없음

무결성 위협:
변경(mod) : 내용 변경, 데이터 순서 변경 등
가장(masquerading) : 신분위장.
재연(Replaying) : 정보획득 -> 추후 재전송
부인(Reputation) : 메세지 송/수신 부인.

가용성 위협:
서비스 거부(DoS) : 정상 사용자가 서비스 이용 못하도록.

---

소극적 공격 : 
정보 획득 but 데이터 변경, 시스템 해 x
기밀성 침해. 
탐지보다 예방. (탐지가 어려워서.)

적극적 공격 : 
데이터 변경, 시스템 위해 가능. 
무결성, 가용성 침해.
방어보다 탐지가 쉬움. (공격루트 많아서)

---

취약점 : 
기,물,관 시스템적 결함. 
공격 위협 대상.
ex) 패치 안된 서버, 약한 비밀번호

위협 :
취약점을 이용해 공격하려는 행위.
뚫으려는 존재. 공격원인들의 집합.
가로채기 / 가로막기 / 변조 / 위조.
자연 / 의도적 / 비의도적.

위험 Risk :
위협이 취약점을 이용했을때 발생하는 손실 가능성.
자산 x 위협 x 취약점.

노출 :
자산이 공격받을 수 있게 드러나 있는 상태

사회공학 :
인간 신뢰 바탕으로 인간 속이기.

principle of weakness link : 보안은 가장 약한 링크보다 강할 수 없다.

---

시점별 통제

예방통제 preventive control: 미리 취약점, 위협 대처
탐지통제 detective contorl : 위협 탐지시 대처. 
교정통제 corrective contorl: 이미 탐지된 위협, 취약점에 대처.감소.

MTTD (Mean Time To Detect)
: 탐지할 때까지 걸린 평균 시간.

MTTR (Mean Time To Respond, Recover)
: 탐지 후 대응 완료까지 걸리는 시간.
